# X 平台规则（Post -> Draft -> 指定 Draft 发布）

本文件是 X 平台（Twitter/X）规则入口。所有产物必须存放在 `item_path`（用户 `note.md` 所在目录）下。

---

## item_path 定义

`item_path` 指向单条内容目录（包含 `note.md`）。

示例：
- `/workspace/mock_obsidian/0001_item2`
- `./mock_obsidian/0001_item2`

---

## 标准目录结构

```text
<item_path>/
  note.md
  X/
    post.md                      # 文案稿（先生成，供用户修改）
    draft/
      0001/
        final.md                 # 完整审阅稿（文字+图片/视频）
        publish.json             # 该 draft 的发布参数
        assets/
          image/
            generated/
              *.png
          video/
            generated/
              *.mp4
```

说明：
- `draft/` 下每个草稿都是独立文件夹。
- 草稿名按创建顺序使用数字：`0001`, `0002`, `0003` ...
- 用户可以让 AI 创建新的 draft 并生成新素材。
- 强制要求（MUST）：每个 `draft/NNNN/final.md` 必须是完整审阅稿，必须包含该草稿发布所需的全部资料（文字、图片、视频、标签）。

---

## 工作流（必须按顺序）

### 1. 先生成文案稿（post）

AI 根据 `note.md` 先生成 `X/post.md`，只做文字内容。

建议格式：

```md
# Tweet
这里是最终推文正文（<=280字符）

# Tags
#标签A #标签B
```

### 2. 用户修改并确认 post.md

- 用户直接编辑 `post.md`
- 用户确认前，不创建真实发布草稿

### 3. 进入 Draft 生成模式

创建新的 `draft/NNNN/`，生成：
1. `draft/NNNN/assets/image/*` 或 `assets/video/*`（真实素材）
2. `draft/NNNN/final.md`（审阅稿）
3. `draft/NNNN/publish.json`（发布参数）

配图可用 Gemini 脚本（写入当前 draft 资产目录）：

```bash
node reference/utils/gemini_image.js --prompt "tweet visual concept ..." --style shinkai --output <item_path>/X/draft/0001/assets/image/generated/gemini_shinkai_001.png
```

`final.md` 示例：

```md
# Tweet
...

# Tags
#标签A #标签B

# Images
![img_1](./assets/image/generated/gemini_shinkai_001.png)

# Videos
- ./assets/video/generated/clip_001.mp4
```

要求：
- `final.md` 不是摘要，而是该 draft 的完整审阅清单与内容展示。
- 所有待发布媒体都必须在 `final.md` 中出现（图片插入、视频路径列出）。

### 4. 发布时必须指定 draft

发布必须显式指定草稿编号：

```bash
node reference/platform/X/scripts/publish_x_post.js <item_path> --draft 0001 --dry-run
node reference/platform/X/scripts/publish_x_post.js <item_path> --draft 0001
```

发布输入来自：
- `X/draft/0001/publish.json`
- `X/draft/0001/assets/*`

## 推文规则

- 单条推文 <= 280 字
- 建议 1-3 个 hashtag
- 发送内容是纯推文文本（不带 Markdown 标题语法）
- 默认使用中文撰写（除非用户明确要求其他语言）

### 文案风格（MUST）

推文要像一个**真实的人在表达想法**——不是营销号在写爆款，也不是 AI 在生成内容。写出来的东西要让人觉得"这是一个有经验的人随手发的"，而不是"这条推文被精心设计过"。

#### 两种都要避免的极端

**❌ AI 味**（机械、空洞、无个性）：
- "在当今社会……"、"随着……的发展"、"值得一提的是"
- "让我们一起来看看"、"接下来我将分享"
- 过度使用"提升"、"赋能"、"助力"等空洞动词
- 全篇没有个人立场，只有正确的废话
- 结尾万能句"你怎么看？欢迎讨论"、"以上就是我的分享，希望对大家有帮助"

**❌ 爆款公式味**（套路感太强、用力过猛）：
- 每条都用"说个可能得罪人的……"、"99%的人不知道……"开头
- 刻意制造冲突和反转来博眼球
- 每句话都像在写广告文案，节奏太紧太满
- 排比句堆砌、每段都工整对仗
- 过度使用数字和对比来制造冲击感

#### 真人感的核心

真人发推的特点是：**有时候结构松散、有时候话说到一半跑题、有时候就是随口一句感慨**。不需要每条推文都是精心打磨的爆款，允许有以下特征：

1. **自然的语气**：像在跟熟人说话，不像在台上演讲。该随意的时候随意，该认真的时候认真。
   - ✅ "最近换了个面霜，意外地好用，分享一下。"
   - ✅ "折腾了一晚上终于搞定了，记录一下免得以后忘。"
   - ❌ "震惊！这款面霜竟然让我的皮肤发生了翻天覆地的变化！"

2. **有干货但不用力**：分享有价值的信息，但不要摆出"我来教你"的姿态。是"我用过觉得不错"，不是"五大秘诀助你逆袭"。
   - ✅ "这个洁面我用了大半年了，氨基酸的，洗完脸不紧绷。敏感肌可以试试。"
   - ❌ "油皮选氨基酸洁面，干皮选奶油质地，敏感肌直接清水洗。三句话省你踩一年的坑。"（太像总结公式了）

3. **允许不完美**：不需要面面俱到，可以只说一个点。真人发推经常就是想到什么说什么，不会每条都做到结构完整。
   - ✅ "突然发现一个事——防晒霜的 SPF 值到了 30 以上差别其实很小，但价格差了好几倍。之前一直在交智商税。"
   - ❌ 把一个小发现硬写成"开头钩子+三段论+有力收尾"的标准结构

4. **个人经历 > 通用建议**：说自己的故事和感受，不要写成通用指南。"我"怎么做的，比"你应该"怎么做更有说服力。
   - ✅ "去年开始把晚上刷手机的时间改成看书，没想到坚持下来了。也没什么诀窍，就是把手机放到另一个房间。"
   - ❌ "想要培养阅读习惯？试试这三个方法：1. 远离手机 2. 设定目标 3. 从感兴趣的书开始"

5. **有观点但不极端**：可以有自己的判断，但不需要每次都"语出惊人"。平实地表达真实想法就够了。
   - ✅ "试了几款国产防晒之后，觉得真不比大牌差。至少日常通勤完全够用了。"
   - ❌ "国产防晒吊打大牌！还在买贵的你就是冤大头！"

#### 示例对比

**话题：平价护肤推荐**

❌ AI 味：
> 今天给大家推荐几款平价护肤品。它们性价比很高，适合学生党和预算有限的朋友。第一款是XX，第二款是XX。希望对大家有所帮助！#护肤 #平价好物

❌ 爆款公式味：
> 说真的，护肤这件事我交过不少学费！后来发现一个规律：贵的不一定好用，但便宜的里面真有宝藏。这三款我反复回购，总价不到200，护肤最怕的就是瞎折腾，找到适合自己的，少就是多。

✅ 真人感：
> 最近朋友问我在用什么护肤品，说了之后她挺惊讶的，因为都不贵。
>
> 其实我以前也买过不少大牌，后来慢慢发现对我来说没太大区别。现在固定用这几样：XX洁面、XX面霜、XX防晒，加起来不到两百。用了快一年了，皮肤状态反而比之前稳定。
>
> 也不是说贵的不好，就是没必要跟风。
> #护肤

**话题：编程经验**

❌ AI 味：
> 学习编程需要掌握正确的方法。建议大家多练习、多阅读源码、多参与开源项目。坚持下去一定会有收获！

❌ 爆款公式味：
> 写了五年代码，最后悔的一件事：前两年花太多时间"学"，太少时间"写"。看十遍教程不如自己从零写一个烂项目！现在带新人我只说一句：别准备好了再开始，开始了你才知道该准备什么。

✅ 真人感：
> 回想了一下自己学编程的过程，走的最大弯路就是看了太多教程。
>
> 当时总觉得要"准备好"再动手，结果一直在看视频、记笔记，真正写代码的时间很少。后来硬着头皮开始写一个小项目，反而进步特别快——因为遇到问题你没法跳过，只能去解决。
>
> 其实哪有什么准备好，写着写着就会了。
> #编程

## 发布前检查清单

1. `X/post.md` 已用户确认
2. 已创建目标 `X/draft/NNNN/`
3. `final.md` 与 `publish.json` 一致，且 `final.md` 已覆盖该 draft 全部资料（文字/图片/视频/标签）
4. `publish.json` 的图片/视频路径指向当前 draft 资产
5. 图片数量 <= 4（X 单条推文限制）
6. `.env` 中 X 凭证完整：
   - `X_API_KEY`
   - `X_API_SECRET`
   - `X_ACCESS_TOKEN`
   - `X_ACCESS_TOKEN_SECRET`

---

## 常见失败排查

- `--draft is required`：发布命令缺少草稿编号
- `publish.json not found`：目标 draft 下缺少发布参数文件
- `Image not found`：路径未指向该 draft 的 `assets/` 目录
- `X supports at most 4 images`：减少单条推文图片数量
- 401/403：检查 token 有效性与 App 权限（Read and Write）
