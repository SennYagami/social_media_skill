# 小红书平台规则（Post -> Draft -> 指定 Draft 发布）

本文件是小红书平台执行规范。AI 执行时必须按本文件流程运行，且所有产物必须落在 `item_path`（用户 `note.md` 所在目录）内。

---

## 1. 作用域与路径

- `item_path`：单条内容目录，必须包含 `note.md`。
- 示例：`/workspace/mock_obsidian/0001_item2`
- 禁止把该条内容的文件写到其他 item 目录。

---

## 2. 目录规范（强制）

```text
<item_path>/
  note.md
  xiaohongshu/
    post.md
    draft/
      0001/
        render.md
        final.md
        publish.json
        assets/
          image/
          video/
      0002/
      ...
```

规则：
- `post.md`：文案阶段文件（先产出，供用户修改确认）。
- `draft/NNNN/`：一次完整草稿产物（可并存多个）。
- `NNNN`：4 位递增数字（`0001`、`0002` ...）。
- 每个 draft 必须自洽，不能依赖其他 draft 的私有素材。

---

## 3. 执行流程（必须按阶段）

### 阶段 A：生成文案稿（只生成文字）

输入：`note.md`

输出：`xiaohongshu/post.md`

建议格式：

```md
# 标题
你的标题（<=20字）

# 封面文字
封面上需要展示的关键文字（用于渲染到封面图上）

# 正文
正文内容...

# Tags
#标签A #标签B #标签C
```

要求：
- 只做文案，不生成最终发布素材。
- 标题、正文、标签结构必须清晰。

#### A.1 标题：核心词精准定位（MUST）

标题必须包含**核心关键词**，精准锁定目标人群，而非泛泛描述。

**原则：**
- 标题中必须出现用户会搜索的核心词（如产品名、痛点词、人群词）。
- 精准定位受众人群，让目标用户一眼识别"这条笔记是写给我的"。
- 避免大而空的泛标题，越具体、越垂直，搜索流量越精准。

**示例对比：**

| 场景 | ❌ 泛标题（受众模糊） | ✅ 核心词标题（精准锁定） |
|------|----------------------|--------------------------|
| 护肤品推荐 | 超好用护肤品大合集 | 敏感肌姐妹必看！超省护肤品合集 |
| 穿搭分享 | 今日穿搭分享 | 小个子女生秋冬穿搭｜显高10cm |
| 美食教程 | 简单好吃的早餐 | 上班族5分钟早餐｜减脂期也能吃 |
| 旅行攻略 | 三亚旅游攻略 | 三亚亲子游攻略｜省钱又好玩 |
| 家居好物 | 好用的收纳好物 | 出租屋改造｜小户型收纳神器 |
| 学习方法 | 高效学习方法 | 考研党必看｜英语从60到85的方法 |

**核心词选择思路：**
1. **人群词**：敏感肌、小个子、上班族、考研党、宝妈……
2. **痛点词**：显高、省钱、减脂、提分、改造……
3. **场景词**：秋冬、出租屋、亲子游、早餐……
4. **效果词**：从XX到XX、N天见效、省XX元……

#### A.2 正文：关键词自然穿插，强化搜索相关性（MUST）

正文必须围绕核心关键词展开，自然穿插相关搜索词，写干货内容。

**原则：**
- 先研究目标用户会搜索什么词，围绕这些词组织内容。
- 关键词要自然融入语境，不能生硬堆砌。
- 内容必须是干货（有信息量、可操作、能解决问题），不能是纯感受流水账。
- 段落开头、小标题、总结处是关键词的黄金位置。

**示例（护肤品推荐）：**
- 核心词：敏感肌、护肤品、平价
- 相关搜索词：泛红、刺痛、屏障修复、换季过敏
- 正文中自然穿插："换季的时候**敏感肌**最容易**泛红刺痛**，这几款**屏障修复**的**平价护肤品**我回购了三次……"

**关键词布局要点：**
1. 第一段（开头）点明主题和核心痛点，命中核心词。
2. 中间段落每段围绕一个分论点，自然带入相关长尾词。
3. 结尾总结呼应标题关键词，引导互动（收藏/关注）。
4. Tags 选择与正文关键词高度一致，补充长尾搜索词。

#### A.3 封面视觉埋词（MUST）

封面图上的文字对笔记的点击率和搜索识别度至关重要。

**原则：**
- 封面必须有醒目文字，突出文章核心内容。
- 封面文字要包含核心关键词，让用户在信息流中快速识别笔记主题。
- 文字要简短有力（通常 6-15 字），优先传达"这篇笔记能给你什么"。
- 封面文字与标题互补，不要完全重复标题。

**封面文字示例：**

| 笔记主题 | 封面文字示例 |
|---------|------------|
| 敏感肌护肤推荐 | 敏感肌急救！3款平价修复好物 |
| 小个子穿搭 | 155小个子｜秋冬显高穿搭公式 |
| 减脂早餐 | 减脂期早餐｜好吃不胖！ |
| 考研英语 | 英语60→85｜我的提分方法 |
| 出租屋改造 | 花500块改造出租屋｜前后对比 |

**在 `post.md` 中的使用方式：**
- 在"# 封面文字"部分写明封面上要展示的关键文字。
- 渲染 `render.md` 封面时，`subtitle` 字段应使用封面文字内容。
- 封面文字需要与标题核心词呼应，但提供更直观的信息吸引点击。

### 阶段 B：用户审阅 post.md

- 用户可修改 `post.md`。
- 未确认前，不进入 Draft 生成。

### 阶段 C：生成 Draft（完整审阅稿 + 素材）

触发条件：用户已确认 `post.md`。

创建新目录：`xiaohongshu/draft/NNNN/`，并生成以下文件：
- `render.md`：用于渲染卡片。
- `assets/image/*`：渲染后图片。
- `final.md`：完整审阅稿。
- `publish.json`：发布参数。

`final.md` 强制要求（MUST）：
- 必须包含该 draft 的全部发布资料：标题、正文、标签、图片、视频（如有）。
- 图片必须用 Markdown 引用该 draft 内路径。
- 必须是“可审阅、可复核、可追溯”的单一入口文件。

`render.md` 强制要求（MUST）：
- 包含 YAML 头：`emoji`、`title`、`subtitle`。
- 正文按分页策略组织（见第 4 节）。

### 阶段 D：发布指定 Draft

- 发布时必须显式指定 draft（例如 `0001`）。
- 发布输入只允许来自目标 `draft/NNNN/`：
  - `publish.json`
  - `assets/image/*`（以及视频，如有）
- 禁止混用其他 draft 的资产。

---

## 4. 分页策略（渲染稳定性）

优先级：手动分段优先，自动切分兜底。

1. 手动分段（推荐）
- 在 `render.md` 用 `---` 按逻辑章节分卡。
- 每段建议 150-250 字，尽量语义完整。

2. 自动切分（兜底）
- 当段落不可控时，允许渲染脚本按高度自动切分。
- 需接受可能出现的语义截断风险。

建议：
- 关键结论、步骤、对比内容优先手动分卡。
- 长列表或超长段落避免放在同一卡片。

## 5. 文件内容模板（最小约束）

### 5.1 render.md 示例

```md
---
emoji: "💡"
title: "主标题"
subtitle: "封面视觉文字（来自 post.md 的「封面文字」，包含核心关键词）"
---

第一页内容...

---

第二页内容...
```

注意：`subtitle` 是渲染到封面图上的文字，必须包含核心关键词，简短有力，与标题互补。

### 5.2 final.md 示例

```md
# 标题
...

# 正文
...

# 图片
![cover](./assets/image/cover.png)
![card_1](./assets/image/card_1.png)

# 视频
- ./assets/video/demo.mp4

# Tags
#标签A #标签B #标签C
```

### 5.3 publish.json 示例

```json
{
  "title": "标题",
  "desc": "正文",
  "images": [
    "./assets/image/cover.png",
    "./assets/image/card_1.png"
  ],
  "videos": []
}
```

---

## 6. 脚本调用（执行层）

1. 渲染图片（示例）

```bash
node reference/platform/xiaohongshu/scripts/render_xhs_v2.js <item_path>/xiaohongshu/draft/0001/render.md -o <item_path>/xiaohongshu/draft/0001/assets/image --style xiaohongshu
```

2. 发布（示例）

```bash
python3 reference/platform/xiaohongshu/scripts/publish_xhs.py --title "标题" --desc "正文" --images <item_path>/xiaohongshu/draft/0001/assets/image/cover.png <item_path>/xiaohongshu/draft/0001/assets/image/card_1.png
```

说明：
- 实际执行前先核对 `final.md` 与 `publish.json`。
- `XHS_COOKIE` 必须有效。

---

## 7. 发布前检查清单

1. `post.md` 已经用户确认。
2. 已创建本次目标 `draft/NNNN/`。
3. `final.md` 已包含全部资料（文字、图片、视频如有）。
4. `publish.json` 中引用路径均指向当前 draft。
5. 所需环境变量已配置且未过期。

---

## 8. 失败处理

- 发布失败优先检查 Cookie 过期。
- 图片缺失优先检查 `assets/image/` 产物是否存在。
- 草稿不满意时新建下一 draft，不覆盖历史 draft。
